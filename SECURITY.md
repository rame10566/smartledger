# Security Policy

## Reporting a Vulnerability

Use GitHub's [Private Vulnerability Reporting](https://github.com/rame10566/smartledger/security/advisories/new)
feature. This keeps the conversation private and gives both of us a
structured workflow.

Please include:
- A description of the vulnerability
- Steps to reproduce
- The affected component (Validation MCP, Ledger MCP, Agent, etc.)
- Your assessment of impact

I will acknowledge receipt within 14 days and provide a timeline for any fix.

## Scope

This is a POC and is not intended for production use without significant
hardening. See `docs/FABRIC-POC-LIMITATIONS.md` for known security and
reliability gaps that are accepted in the POC scope.

## Supported Versions

Only the `main` branch is supported. Older tags and branches receive no
security updates.
